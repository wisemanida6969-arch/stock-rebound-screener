# PC와 연결 없이 · 모바일만으로 가동

집 PC를 켜 두거나 같은 Wi-Fi로 묶을 필요 **없이**, **폰만**으로 쓰려면 아래 중 **한 번만** 하면 됩니다.  
(시세 데이터는 인터넷(LTE/Wi-Fi)만 있으면 됩니다.)

---

## 핵심

| 단계 | 내용 |
|------|------|
| **1회** | `docs` 폴더 안 파일을 **무료 웹 호스팅**에 올림 → **고정 주소** 생김 |
| **이후** | 그 주소만 폰 브라우저(또는 홈 화면 앱)로 열면 됨 · **PC 불필요** |

앱은 **HTML 한 장**이라 서버 프로그램·Streamlit·집 PC **전부 없음**.

---

## 방법 A — Netlify Drop (가입만, 가장 빠름)

1. PC **또는** 폰에서 [app.netlify.com/drop](https://app.netlify.com/drop) 접속 (간단 가입).
2. **`docs` 폴더 안 내용만** ZIP으로 압축  
   - ZIP 안에 **`index.html`이 바로 보이게** (폴더 한 겹 더 넣지 말 것).
3. ZIP을 페이지에 **드래그 앤 드롭**.
4. 나오는 **`https://xxxx.netlify.app`** 주소를 **폰 즐겨찾기** 또는 **홈 화면에 추가**.

→ 이후에는 **어디서든 폰만**으로 접속. PC 끄고 나서도 동일 주소로 계속 사용.

---

## 방법 B — GitHub Pages (주소 예쁘게 유지)

1. GitHub에 저장소 만들고 이 프로젝트 푸시.
2. **Settings → Pages → Branch `main`, Folder `/docs`** 저장.
3. 주소: `https://아이디.github.io/저장소이름/`

→ PC 없이 폰에서만 접속 가능.

---

## 방법 C — Cloudflare Pages

저장소 연동 또는 폴더 업로드 후 배포 → 발급된 `*.pages.dev` 주소를 폰에 저장.

---

## 폰에서 “앱처럼” 쓰기

- **iPhone**: Safari → 공유 → **홈 화면에 추가**
- **Android**: Chrome → **홈 화면에 추가** / **앱 설치**

`manifest.json` 때문에 전체 화면으로 뜨는 경우가 많습니다.

---

## 정리

- **PC 연결 없이 모바일 단독** = **한 번 배포한 HTTPS 주소**만 있으면 됨.
- 배포 후에는 **Wi-Fi든 LTE든** 그 주소만 열면 되고, **집 PC는 완전히 무관**합니다.

문제 생기면 `docs/index.html` · `manifest.json` · `sw.js` 세 파일이 같은 주소에 같이 있어야 합니다.

---

## 「실패 · 티커·네트워크」가 뜰 때

브라우저 → Yahoo 직접 접속이 막혀 있어 **공개 CORS 프록시**를 여러 개 돌리고 있습니다. 프록시가 막히면 같은 메시지가 납니다.

1. **Wi‑Fi로 바꿔서** 다시 **실시간 차트** 누르기  
2. **잠시 뒤** 재시도 (일시 과부하)  
3. 티커는 **영문 대문자** (예: `AAPL`)  
4. **VPN** 켜 보거나 끄기  
5. 그래도 안 되면 당일은 프록시 전체가 불안정할 수 있음 → **PC에서 Streamlit**(`streamlit run app.py`)은 Yahoo를 서버에서 받아서 상대적으로 잘 됨
